# 🚨 Важная информация о текущем состоянии приложения

## Проблемы с Expo Go / Development режимом

### ❌ Известные проблемы:

1. **ERR_NAME_NOT_RESOLVED** - Expo сервер работает в локальной сети, но недоступен из-за конфигурации контейнера
2. **React Native Web FlatList Bug** - Список чатов не отображается в Expo Go на web/Android из-за бага с FlatList

### ⚠️ Почему не работает Expo Go?

Приложение разрабатывается в Docker контейнере, который:
- Не имеет прямого доступа к локальной сети
- Expo tunnel (ngrok) не работает стабильно
- LAN mode не доступен для внешних устройств

**React Native Web имеет критический баг с FlatList**, который мешает отображению списков на всех платформах в dev режиме.

## ✅ РЕШЕНИЕ: Сборка Production APK

Приложение **полностью функционально** при сборке в production режиме через EAS Build.

### Быстрая сборка APK для тестирования:

```bash
# 1. Установите EAS CLI (если ещё не установлен)
npm install -g eas-cli

# 2. Авторизуйтесь в Expo
eas login

# 3. Инициализируйте проект
cd /app/frontend
eas init

# 4. Соберите APK для тестирования
eas build --platform android --profile preview --local
```

**Время сборки:** ~20-30 минут

### Альтернатива: Облачная сборка (рекомендуется)

```bash
# Сборка в облаке (не требует мощного компьютера)
eas build --platform android --profile preview
```

**Время сборки:** ~15-20 минут
**Результат:** Ссылка на скачивание APK

## 📱 Установка APK на устройство

После получения APK файла:

2. Откройте файл

## 🔧 Если хотите попробовать dev режим (не рекомендуется)

### Требования:
- Телефон и компьютер в одной Wi-Fi сети
- Expo сервер запущен в tunnel mode
- Но даже это не гарантирует работу из-за бага FlatList

### Команды:
```bash
cd /app/frontend
npx expo start --tunnel
# Отсканируйте QR-код в Expo Go
```

**⚠️ ВАЖНО:** Даже при успешном подключении список чатов НЕ отобразится из-за React Native Web FlatList bug. Это ограничение Expo в dev режиме.
